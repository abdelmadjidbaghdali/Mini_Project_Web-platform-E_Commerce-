## 🌐 Préambule — Pourquoi ce projet ?

Les plateformes e-commerce sont parmi les **applications Web les plus complexes et les plus utilisées** au monde.

Elles combinent :

- Gestion des utilisateurs
- Sécurité
- Persistance des données
- Logique métier
- Performance
- Expérience utilisateur

Ce mini-projet vous place dans une **situation proche d’un contexte professionnel réel** : concevoir une **application e-commerce complète**, depuis la base de données jusqu’à l’interface utilisateur, en respectant les **principes d’architecture Web modernes**.

# 🎯 Objectifs pédagogiques

À l’issue de ce mini-projet, vous serez capables de :

- Concevoir une architecture Web complète (Frontend / Backend)
- Implémenter une logique métier e-commerce
- Gérer l’authentification et les rôles utilisateurs
- Manipuler les cookies et les sessions
- Interagir avec une base de données

# 📌 Principe général

Chaque équipe doit développer un **site Web e-commerce fonctionnel**, permettant la vente d’un type d’item librement choisi :

- 📚 Livres
- 💻 Produits électroniques
- 👕 Vêtements
- ⚽ Équipements sportifs
- Ou tout autre type de produit

---

# 👥 Travail en équipe & Git

- Projet réalisé en équipe
- Chaque équipe crée un dépôt GitHub nommé :

```text
pravan_2025_team_X
```

où **X** correspond au numéro de l'équipe.

# 🧩 Fonctionnalités obligatoires

## 🏪 Boutique

- Affichage des produits
- Chargement depuis MySQL
- Nom
- Description
- Prix

---

## 🔍 Recherche & filtrage

- Recherche par mot-clé
- Filtre par catégorie
- Filtre par prix
- Autres critères

---

## 📄 Détail d'un produit

- Description complète
- Prix
- Ajouter au panier

---

## 👤 Authentification

- Connexion
- Déconnexion
- Gestion des sessions
- Cookies

---

## 🛍️ Panier

- Ajouter un produit
- Supprimer un produit
- Modifier les quantités
- Calcul du total

---

## 🔐 Administration

L'administrateur peut :

- Ajouter un produit
- Modifier un produit
- Supprimer un produit

L'accès doit être sécurisé.

---

# 🗄️ Base de données

La base de données doit contenir au minimum :

- `users`
- `items`
- `cart`
- Gestion des rôles (`user`, `admin`)

Le schéma doit être documenté et justifié.
